About Semantic HTML

Semantic HTML introduces meaning to the web page rather than just presentation. For example, a <p> tag indicates that the enclosed text is a paragraph. This is both semantic and presentational because people know what paragraphs are, and browsers know how to display them.

On the flip side of this equation, tags like <b> and <i> are not semantic. They define only how the text should look (bold or italic), and don't provide any additional meaning to the markup.

Benefits of Semantic HTML

Using semantic HTML helps search engines and other user devices determine the importance and context of web pages. The semantic elements clearly describe its meaning to both the browser and the developer.

Additionally, semantic HTML is much easier to read and helps with:

  • SEO (Search Engine Optimization)
  • Accessibility
  • Maintainability
  • Developer experience

Did You Know?

HTML5 introduced many semantic elements like <header>, <footer>, <article>, <section>, <nav>, <aside> and many more to help structure the document in a more meaningful way.
